  What it is

  A Maven + Java 17 Selenium/TestNG test-automation framework by Pramod Dutta
  (The Testing Academy). It's primarily a teaching/demo framework вҖ” it shows
  multiple styles of writing UI tests against demo web apps, mainly app.vwo.com
  login flows.

  How it's built

  - Build/run: Maven (mvn clean test), TestNG suites, Allure reporting, Log4j2
  logging, Apache POI for Excel-driven data.
  - Java 17, Selenium 4.27.0, TestNG 7.10.2 (versions centralized in pom.xml).
  - Supports local, headless, and remote (Selenoid/Docker grid) execution.

  Architecture (the big picture)

  - DriverManager вҖ” thread-safe ThreadLocal<WebDriver>; builds
  chrome/firefox/edge drivers locally or as RemoteWebDriver. All driver access
  goes through this.
  - CommonToAllTest (base test) вҖ” every test extends it; @BeforeMethod inits the
  driver, @AfterMethod screenshots-on-failure into Allure then quits.
  Per-method lifecycle вҶ’ safe method-level parallelism.
  - Two page-object styles, side by side for contrast: classic POM
  (pages/POM/..., By locators + constructor-injected driver) and Page Factory
  (pages/PF/..., @FindBy + extends CommonToAll).
  - Listeners вҖ” global retry (RetryListener force-applies RetryAnalyzer to every
  test) + screenshot listener.

  Things worth knowing (gotchas I flagged in CLAUDE.md)

  - data.properties is the real config source вҖ” not -D flags. DriverManager
  reads browser/headless/mode only from that file, so Maven profiles and CI
  -Dbrowser=... flags are effectively ignored by the driver.
  - Tests must run from the repo root (files resolved via user.dir + hardcoded
  relative paths).
  - The smoke/regression Maven profiles point at TestNG files that don't exist;
  only the two testng_vwo_*_part4.xml suites are real.
  - VWO tests are numbered 01вҶ’04 to show a progression of techniques (plain вҶ’
  properties вҶ’ retry вҶ’ screenshot).
  - Needed git config core.longpaths true to clone on Windows (long filenames).

  Current state
